# LeetCode Master Skill

本 Skill 旨在帮助用户深度掌握 LeetCode 热题 100 的核心算法，并具备生成高质量原创练习题的能力。

## 工作流程

### 1. 考点总结与查询
当用户询问某类算法或特定题目时：
- 阅读 `references/hot100.md` 获取标准考点。
- 总结核心逻辑，并提供 2-3 个代表性题目编号。
- 解释该考点在面试中的常见变形。

### 2. 交互式题目生成
当用户表达“希望出一些原创题目”时，**必须**按以下步骤进行交互：

**第一步：需求调研**
询问用户以下信息（可一次性询问，也可逐步引导）：
1. **目标难度**：简单 (Easy), 中等 (Medium), 困难 (Hard)。
2. **考点偏好**：例如“双指针”、“动态规划”或“模拟 LeetCode 热题 100 中的某类题目”。
3. **应用场景**：例如“模拟面试”、“日常练习”、“竞赛模拟”。

**第二步：题目设计**
根据用户反馈，设计一道**原创**题目。要求包括：
- **题目描述**：背景设定新颖，逻辑清晰。
- **示例输入/输出**：至少提供两个清晰的用例。
- **核心考点**：明确指出该题参考了热题 100 中的哪些知识点。
- **解题提示**（初稿隐藏，仅在用户要求时提供）。

**第三步：反馈与调整**
询问用户题目难度是否合适，是否需要提供代码参考或进一步讲解。

## 资源引用
- 所有热题 100 的数据存储在 `references/hot100.md` 中。
- 在生成题目时，应确保逻辑严密且具有 LeetCode 风格。
